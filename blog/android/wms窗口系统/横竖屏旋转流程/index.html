横竖屏旋转流程

2024年6月1日 / android / wms窗口系统
androidWMS屏幕旋转Configuration

简介

屏幕旋转是 Android 系统中一个复杂的流程，涉及多个系统组件的协作。

触发条件

  1. 传感器检测：加速度传感器检测设备方向变化
  2. 应用请求：应用设置屏幕方向
  3. 系统设置：用户锁定屏幕旋转

旋转流程

1. 方向变化检测

OrientationListener.onSensorChanged()
    → DisplayRotation.updateOrientation()

2. 配置更新

DisplayContent.updateConfiguration()
    → ConfigurationController.updateConfiguration()
    → ActivityThread.handleConfigurationChanged()

3. 窗口重建

ActivityThread.handleRelaunchActivity()
    → Activity.onDestroy()
    → Activity.onCreate()
    → 窗口重建

4. 动画执行

WMS.startFreezingDisplay()
    → 播放旋转动画
    → WMS.stopFreezingDisplay()

关键类

类名职责
DisplayRotation管理屏幕旋转逻辑
DisplayContent显示内容管理
WindowAnimator执行旋转动画
ActivityRecord记录 Activity 状态

配置变化处理

不重建 Activity

<activity android:configChanges="orientation|screenSize">

手动处理

@Override
public void onConfigurationChanged(Configuration newConfig) {
    super.onConfigurationChanged(newConfig);
    // 手动处理配置变化
}

旋转动画

  1. 截图：保存当前屏幕内容
  2. 缩放：执行缩放动画
  3. 切换：切换到新的屏幕方向
  4. 恢复：显示新的界面内容

常见问题

  1. 旋转卡顿：Activity 重建耗时
  2. 状态丢失：未正确保存和恢复状态
  3. 动画异常：旋转动画执行失败

总结

屏幕旋转涉及方向检测、配置更新、窗口重建和动画执行等多个环节，需要各组件协同工作。
