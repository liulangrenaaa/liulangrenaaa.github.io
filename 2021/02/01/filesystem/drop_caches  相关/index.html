drop_caches 相关

proc文件系统

drop_caches 文件位于系统 /proc/sys/vm

可以写入1:drop page cache 2: drop slab 3: drop page cache and slab 等值

相关代码主要在 fs/drop_caches.c 目录下

如果写入的值 bit1 是1，就会回收page cache，如果写入的值 bit2 是1，就会回收slab。

pagecache 回收

简化一些定义 lock操作，如下：

首先遍历super_blocks，然后 遍历sb->s_inodes，实际的回收 pagecache是针对于 inode->i_mappings的，对应代码就是 invalidate_mapping_pages(inode->i_mapping, 0, -1);。

invalidate_mapping_pages 会尝试 将这个inode对应的pagecacheinvaild掉（就像注释里面写的，除去dirty locked writeback的 page）。

所以 drop cache 不会将脏页给drop掉，要想 drop掉脏页，需要先sync()一下。

slab 回收

drop_slab 过程首先会遍历 online_node，对该 node进行 drop_slab_node。

在drop_slab_node 中，会遍历 memcg进行shrink_slab，退出条件是，这一次回收的 slab objects 小于10.

在 shrink_slab 中，会遍历注册的 shrinker，使用do_shrink_slab 对每个shrinker 进行slab内存回收。

在 do_shrink_slab 中，有两个 tracepoint点，分别是 start end，可以用来追踪相关事件，
实际回收的函数是 shrinker->scan_objects(shrinker, shrinkctl);

不同的slab，会注册不同的 shrinker.
