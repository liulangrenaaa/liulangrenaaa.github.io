KASAN 原理分析

原理

granule size == 8
tags num == 2^8 == 256

代码实现分析

tag 有哪些，怎么产生的

kernel 认为 0xFF 的tag 都是有效的，将不可访问的内存都标记为 0xFE，认为是无效的。
在不使能 CONFIG_KASAN_HW_TAGS 特性下，最小的 TAG是 0x00；使能情况下（只有aarch64支持），最小的 TAG是 0xF0。

通过 kasan_random_tag 可以随机产生一个tag val
