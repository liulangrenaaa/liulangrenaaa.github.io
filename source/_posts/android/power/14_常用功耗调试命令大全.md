# 常用功耗调试命令大全

在进行 Android 功耗优化、问题排查及自动化脚本编写时，熟练使用 `adb` 命令行工具是功耗工程师的基本功。通过这些命令，我们可以模拟充电状态、强行切换 Doze 模式、重置电量统计、模拟发热等级，甚至可以直接观察内核唤醒锁。

本文为你整理了一份 **Android 14 (AOSP 14) 最完整的功耗调试命令“百宝箱”**。

---

## 一、 电源管理核心服务 (PowerManagerService) 命令

用于查询当前系统整体电源状态、Wakefulness 状态机以及正在持有的锁。

### 1. 查看 PMS 全量状态
```bash
adb shell dumpsys power
```
> **输出关键解析**：
> * `mWakefulness=Interactive`：当前屏幕处于完全点亮亮屏状态。
> * `mHoldingWakeLockSuspendBlocker=true`：当前由于 WakeLock 或亮屏持有底层锁，阻止休眠。
> * `Wake Locks: size=1`：列出当前全系统所有正在被持有的 Java WakeLock 的 UID、Tag 和申请时间。

### 2. 精准过滤当前持有的 WakeLock
```bash
adb shell dumpsys power | grep -A 10 "Wake Locks:"
```

## 二、 电池状态模拟器 (Battery Service) 命令

在办公室开发时，手机通常插着 USB 数据线（处于充电状态），这会导致许多后台限制（如 Doze 模式、JobScheduler 延迟）无法正常触发。我们可以使用以下命令**断开虚拟充电链路并修改电池参数**。

### 1. 强行模拟断开充电（Unplug）
```bash
adb shell dumpsys battery unplug
```
* **效果**：系统会认为当前拔掉了充电器（即使物理上依然插着 USB 数据线供电并保持 ADB 连接），以测试后台省电逻辑。

### 2. 强行修改电池电量百分比
```bash
adb shell dumpsys battery set level 15
```
* **效果**：将系统显示的电量强制改为 15%，触发低电量省电提醒。

### 3. 强行修改电池充电状态
```bash
# status 对应：1=Unknown, 2=Charging, 3=Discharging, 4=Not Charging, 5=Full
adb shell dumpsys battery set status 3
```

### 4. 强行修改电池温度 (单位: 0.1℃)
```bash
adb shell dumpsys battery set temp 460
```
* **效果**：模拟电池达到 46℃ 高温，测试温控降电流和警告逻辑。

### 5. 查看当前电池详细信息
```bash
adb shell dumpsys battery
```

### 6. 【最重要】重置电池模拟器（恢复物理读取）
> [!CAUTION]
> 测试完毕后，**必须运行此命令**，否则手机将永远保持你设定的模拟参数，电池百分比将不再更新，甚至充不进电！
```bash
adb shell dumpsys battery reset
```

## 三、 Doze (低电耗模式) 强行流转命令

无需静置手机数小时，使用命令即可瞬间控制 Doze 状态机。

### 1. 强行模拟进入 Deep Doze 状态
```bash
# 第一步：断开充电
adb shell dumpsys battery unplug
# 第二步：关闭屏幕
adb shell input keyevent 26
# 第三步：强行让 Doze 状态机流转到 IDLE
adb shell dumpsys deviceidle force-idle deep
```

### 2. 逐步流转 Doze 状态机
```bash
# 每次运行此命令，状态机会向前挪动一个状态 (如从 ACTIVE -> INACTIVE -> SENSING -> IDLE)
adb shell dumpsys deviceidle step deep
```

### 3. 退出强行 Doze 状态
```bash
adb shell dumpsys deviceidle unforce
```

### 4. 管理 Doze 白名单
```bash
# 查看当前所有的白名单应用
adb shell dumpsys deviceidle whitelist
# 将指定应用临时加入白名单
adb shell dumpsys deviceidle whitelist +com.example.app
# 将指定应用移出白名单
adb shell dumpsys deviceidle whitelist -com.example.app
```

## 四、 应用待机群组 (App Standby Buckets) 控制

测试单个应用在不同活跃度群组下的后台表现。

### 1. 查询目标应用当前所处的群组
```bash
adb shell am get-standby-bucket <packageName>
```
> **返回值对应关系**：
> `10` = ACTIVE | `20` = WORKING_SET | `30` = FREQUENT | `40` = RARE | `45` = RESTRICTED

### 2. 强行修改目标应用的群组
```bash
# 将应用打入“受限”群组，体验最严苛的 Job 拦截
adb shell am set-standby-bucket <packageName> restricted
# 恢复活跃状态
adb shell am set-standby-bucket <packageName> active
```

## 五、 电池历史账单 (BatteryStats) 重置与导出

在进行特定场景耗电测试前，必须清理旧的统计缓存，防止之前的累加数据干扰测试结果。

### 1. 重置 BatteryStats 历史账单
```bash
adb shell dumpsys batterystats --reset
```
* **使用场景**：在手机满电拔下那一刻，运行此命令，开始纯净耗电测试。

### 2. 将电量统计缓冲区立刻写入磁盘
```bash
adb shell dumpsys batterystats --write
```

### 3. 导出全量 BatteryStats 报告（供 Battery Historian 解析）
```bash
adb shell bugreport > bugreport.zip
# 或者在较旧版本上：
adb shell dumpsys batterystats > batterystats.txt
```

## 六、 温控管理器 (Thermal Service) 欺骗命令

用于测试手机在高热环境下的系统策略变化（如 60Hz 降帧、降亮屏、限频）。

### 1. 强行覆盖整机温控严重等级
```bash
# 强行模拟等级 3 (SEVERE)，观察屏幕是否从 120Hz 降至 60Hz，大核是否限频
adb shell cmd thermalservice override-status 3
```

### 2. 重置温控状态（恢复真实物理传感器读取）
```bash
adb shell cmd thermalservice reset
```

## 七、 内核唤醒源查看命令

必须在**已 Root** 的调试设备上使用：

```bash
# 查看当前占用时间最长的内核唤醒源 (对于 4.x 以上内核)
adb shell cat /sys/kernel/debug/wakeup_sources
# 或使用快捷路径 (部分 ROM 映射在 /d/ 下)
adb shell cat /d/wakeup_sources
```
