## 模板一：Android 整机待机与续航功耗深度调优项目 (ROM 级别)

### 1. 项目基本信息
* **项目名称**：旗舰手机平台待机与整机续航功耗专项优化
* **担任角色**：资深功耗调优工程师 / Android Framework 性能专家
* **项目描述**：针对基于 Android 14（AOSP 14）定制的旗舰机型开展整机功耗攻坚，着力解决设备在夜间灭屏待机电流偏高、系统休眠跳变频繁（Suspend Aborts）以及高负载（游戏、相机）下的温控卡顿发热问题，打通软硬件一体化的能效优化。

### 2. 工作职责与技术细节描述（可直接精简并写入简历）
* **系统级待机电流攻坚**：
  使用程控稳压电源物理监测待机电流，配合 **`Battery Historian`** 和 **`Perfetto`** 诊断灭屏后台耗电。通过重构 **`DeviceIdleController` (Doze 模式)** 的传感检测逻辑，优化 **`AlarmManagerService`** 对齐唤醒窗口算法，大幅合并后台零散唤醒。
* **解决内核级休眠中止 (Suspend Abort)**：
  深入 Linux 内核层，读取分析 **`/sys/kernel/debug/wakeup_sources`**，精确定位并修复了 20 余处由于 Wi-Fi 驱动多播过滤失效、接近传感器 I2C 中断抖动导致的内核休眠打断故障。
* **EAS (能量感知调度) 与性能优化**：
  协同芯片厂商调整 **`EAS` 能量模型 (Energy Model)** 的功耗参数，优化 **`schedutil` 调频器** 的负载响应延迟，实现小任务在小核集群的高能效绑定，成功减少超大核（Prime Core）在普通阅读场景下的无谓拉高。
* **温控降退与充电安全设计**：
  重构 **`ThermalManagerService`** 在 `SEVERE` 与 `CRITICAL` 等级下的温控降退曲线，实现了与 `DisplayPowerController` 亮帧动态节流及大核 CPU 动态限频的联动，在维持视觉顺畅的前提下，解决了游戏场景下的瞬时过热降频问题。

### 3. 量化核心成果（用数字征服面试官）
* 成功将设备的静态待机电流**从 5.2mA 压低至 2.5mA（降幅达 52%）**，夜间纯待机（8小时）耗电控制在 **2% 以内**。
* 解决了 100% 的 Suspend-Resume 挂起失败循环，使系统成功深睡（C2/C3 状态以上）占比提升至待机时段的 **94% 以上**。
* 优化后，整机典型场景续航时间（DoU）**提升了 1.8 小时**，极大地改善了产品的用户口碑。

## 模板二：Android 亿级日活 App 功耗治理与 APM 监控系统 (应用级别)

### 1. 项目基本信息
* **项目名称**：亿级日活应用功耗治理专项与 APM 功耗监控底盘建设
* **担任角色**：APM 性能监控负责人 / 客户端资深架构师
* **项目描述**：由于 App 业务线众多、后台任务杂乱，导致线上电量口碑下降。本项目旨在建立一套线上自动化功耗异常捕捉 APM 系统，并对后台 WakeLock 泄露、高频定位、高负载软解码等耗电顽疾进行专项治理。

### 2. 工作职责与技术细节描述（可直接精简并写入简历）
* **设计高精度 APM 功耗监控 SDK**：
  利用 **`ASM / AspectJ` 字节码插桩技术**，在编译期无侵入地对 `PowerManager.newWakeLock()`、`AlarmManager.set()`、`LocationManager.requestLocationUpdates()` 等高敏功耗 API 进行 Hook 拦截，自动在线上捕获申请者的调用堆栈、持有时间并进行聚类分析。
* **建立 WakeLock 死亡防线与泄露拦截**：
  针对线上后台 WakeLock 泄露问题，封装了带超时机制的锁代理，并在 APM 中实现进程生命周期关联绑定。一旦进程被 Cached 或处于后台，强行释放或冻结所有残留的 Partial WakeLock。
* **射频网络与定位功耗治理**：
  重构后台轮询和服务，将原有的高频心跳包全面迁移至 **`JobScheduler`** 并强制配置“连 Wi-Fi 且充电”约束；在定位场景中，通过引入 **FLP (融合定位)** 替代高耗电的 `GPS_PROVIDER`，结合传感器数据实现高能效的离散位置获取。
* **音视频硬解能效跃升**：
  优化播放器底层解码机制，全面废弃 CPU 软件解码，建立 `MediaCodec` 硬件解码自适应降退策略，将视频渲染负载完全卸载至 Soc 的 VDEC 专用硅片上，极大地降低了播放过程中的 CPU 开销。

### 3. 量化核心成果（用数字征服面试官）
* 线上 APM 系统成功部署后，**首月帮助业务团队定位并彻底消除了 98% 的后台 WakeLock 泄露故障**。
* 通过硬解替代和 Job 合并发送，将视频播放状态下的整机物理电流**从 460mA 削减至 220mA（功耗降低 52%）**，彻底解决了播放视频手感发热的问题。
* 使得应用后台纯待机下的 CPU 占比（Background CPU Occupancy）**减少了 68%**，因耗电引起的系统后台强杀率（AOM Kill Rate）**下降了 40%**。
