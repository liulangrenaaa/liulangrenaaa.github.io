# 电池统计与耗电归因机制

当我们在手机“电池设置”中查看哪个应用最耗电，或者使用 `BatteryHistorian` 工具分析耗电波形时，这些极其详尽的数据都是由 Android 系统的 **`BatteryStatsService`** 收集并计算出来的。

本节我们将基于 **AOSP 14**，剖析 Android 电池统计体系的核心设计、传统的 `power_profile.xml` 估算模型、以及现代 Android 引入的精细化物理电量轨测量（ODPM / PowerStats HAL）机制。

---

## 一、 电池统计大管家：`BatteryStatsService`

`BatteryStatsService`（运行在 `system_server` 中）是电量统计的总入口。它的底层核心是 **`BatteryStatsImpl`**，该类以极细粒度记录了系统自开机或充电以来所有的软硬件电量状态变更。

### 1. 数据收集机制：主动上报与被动监听
为了能够精确给每个 UID 记账，`BatteryStatsImpl` 采用了双向的数据收集策略：

* **主动上报（Active Callback）**：
  当应用申请锁、开启扫描时，对应的服务会直接通过 `Notifier.java` 主动向 `BatteryStats` 打卡。
  * 比如：当 PMS 申请 WakeLock 时，会调用 `mBatteryStats.noteStartWakeLocked(...)`。
  * 当定位服务获取 GPS 时，调用 `mBatteryStats.noteStartSensorLocked(...)`。

* **拉取更新（Pull Mechanism）**：
  对于像 CPU、Wi-Fi、蓝牙、蜂窝移动网络等模块，数据流量和状态频繁变化。`BatteryStatsImpl` 会注册定时任务（每隔一段时间）或者在特定事件（如屏幕亮灭屏、拔掉充电器）触发时，主动去对应的驱动或控制器（Controller）拉取数据。
  * 例如：在灭屏时调用 `updateCpuTimeLocked()` 拉取所有 UID 消耗的 CPU 时间片（读取 `/proc/uid_cputime/` 节点）。

## 二、 经典电量估算模型：`power_profile.xml`

在早期的 Android 系统（以及目前许多没有安装独立物理电量计的设备）中，系统本身**无法得知每个硬件模块真实的耗电物理电流**。

为了估算耗电，Android 采用了一套 **“经验常数模型”**：
$$\text{估计耗电量 (mAh)} = \sum (\text{硬件模块工作时长} \times \text{硬件额定电流常数})$$

### 1. `power_profile.xml` 配置
这套模型所需的额定电流常数，存放在系统资源文件的 `frameworks/base/core/res/res/xml/power_profile.xml` 中。每个 OEM 厂商在适配一款新手机时，必须在实验室测量出各个硬件模块的电流，并写入该文件：

```xml
<!-- 某设备 power_profile.xml 简化片段 -->
<device name="Android">
  <!-- 屏幕在最大亮度下的额外电流 (mA) -->
  <value name="screen.full">350.0</value>
  <!-- 屏幕亮屏时的基础维持电流 (mA) -->
  <value name="screen.on">120.0</value>
  <!-- 蓝牙在接收/传输数据时的电流 (mA) -->
  <value name="bluetooth.active">15.5</value>
  <!-- Wi-Fi 扫描时的电流 (mA) -->
  <value name="wifi.scan">80.0</value>
  <!-- CPU 核心 0 在不同频率下的电流 (mA) -->
  <array name="cpu.active.cluster0">
    <value>12.5</value>  <!--  300MHz -->
    <value>25.8</value>  <!--  900MHz -->
    <value>45.2</value>  <!-- 1500MHz -->
  </array>
</device>
```

### 2. 电量计算器：`PowerCalculator`
在 AOSP 14 中，电量的具体归因计算被模块化封装在了一系列的 `PowerCalculator` 子类中：
* **`CpuPowerCalculator`**：根据应用在各 CPU 核心、各频率下的执行时间片（Time-in-state），乘上 `cpu.active` 对应的电流参数，算出 CPU 耗电。
* **`ScreenPowerCalculator`**：根据屏幕点亮时间、亮度等级，乘以 `screen.on` 与 `screen.full` 算出显示耗电。
* **`WifiPowerCalculator` / `BluetoothPowerCalculator`**：计算射频芯片处于 RX/TX 状态的时长，乘上对应射频常数。

## 三、 现代高精度归因：ODPM 与 `PowerStats HAL`

虽然 `power_profile.xml` 简单易用，但在现代复杂的异构芯片（SoC）和多路供电架构下，它的误差非常大（有时可达 20% 以上）。

为了解决这个问题，自 Android 10 起引入、并在 **Android 14** 中高度成熟了 **ODPM（On-Device Power Monitor，在设电源监控）** 机制。

```
              ┌─────────────────────────────────────┐
              │          BatteryStatsImpl           │
              └──────────────────▲──────────────────┘
                                 │ Binder IPC
              ┌──────────────────┴──────────────────┐
              │          PowerStatsService          │
              └──────────────────▲──────────────────┘
                                 │ AIDL Call
              ┌──────────────────┴──────────────────┐
              │           IPowerStats HAL           │
              └──────────────────▲──────────────────┘
                                 │
              ┌──────────────────┴──────────────────┐
              │ 硬件物理传感器: 各种电压/电流轨 (Rails)  │
              │  (如: GPU Rail, Modem Rail, CPU Rail)│
              └─────────────────────────────────────┘
```

### 1. `IPowerStats` HAL 物理读取
现代手机的电源管理芯片（PMIC）中集成了解调器级的**库仑计（Coulomb Counter）**，可以物理测量不同电源轨（Power Rails）上通过的实时电压和电流。
* 厂商通过实现 `IPowerStats` HAL，将这些硬件电源轨的数据暴露出来。
* `PowerStatsService` 定期轮询这些接口，直接获取以 **微焦耳（micro-joules, $\mu J$）** 为单位的硬件子系统累计消耗的真实物理能量值。

### 2. 物理与经验的结合
在 Android 14 中，`BatteryStats` 将物理 Rails 测出的精准总能量，与应用持有的 CPU 时间片、WakeLock 频次进行按比例权重分摊。
这使得电量统计不仅能准确得出“这半小时内 Wi-Fi 芯片物理消耗了 500 焦耳能量”，还能将其精确平摊到具体是哪一个 UID 发起了网络请求。

## 四、 耗电归因计算器架构（AOSP 14 核心类图）

当你调用 `dumpsys batterystats` 时，系统在底层会触发以下计算逻辑：

```java
// 核心调用链简化
BatteryUsageStatsProvider provider = new BatteryUsageStatsProvider(mContext, mPowerProfile);
BatteryUsageStats stats = provider.getBatteryUsageStats(query);
```

其底层包含了如下计算器家族：

```
                    ┌─────────────────────────┐
                    │     PowerCalculator     │ (抽象基类)
                    └────────────┬────────────┘
                                 ├──> CpuPowerCalculator
                                 ├──> ScreenPowerCalculator
                                 ├──> WifiPowerCalculator
                                 ├──> BluetoothPowerCalculator
                                 └──> MobileRadioPowerCalculator
```

每一个 Calculator 都实现 `calculate(List<BatterySipper>...)` 方法，负责将统计结果封装到 `BatteryUsageStats` 对象中，最后输出给系统设置或调试工具。
