# 充电与电池服务架构

在 Android 设备中，负责监控电池电量变化、电池温度、电压，以及控制充电逻辑的服务是 **`BatteryService`**。它与系统底层的 Health HAL（健康抽象层）以及内核的 `power_supply` 驱动密切配合，共同守护电池的安全与寿命。

本节我们将基于 **AOSP 14**，从底层驱动、HAL 接口到 Java 层服务，梳理完整的电池服务与充电管理架构。

---

## 一、 整体技术架构图

Android 电池服务的架构由底向上分为四层：

```
┌────────────────────────────────────────────────────────────────────────┐
│ 1. Java 框架与系统应用 (System Server & Apps)                            │
│    - BatteryService (发布广播 ACTION_BATTERY_CHANGED)                   │
│    - SystemUI (更新状态栏电池百分比及充电动画)                             │
└───────────────────────────────────┬────────────────────────────────────┘
                                    │ Binder IPC
┌───────────────────────────────────▼────────────────────────────────────┐
│ 2. 硬件抽象层 (Health HAL - android.hardware.health)                    │
│    - healthd 守护进程 (提供 IHealth.aidl 服务)                          │
│    - 读取并格式化底层 Sysfs 电池节点数据                                 │
└───────────────────────────────────┬────────────────────────────────────┘
                                    │ Sysfs File Reading / Netlink Event
┌───────────────────────────────────▼────────────────────────────────────┐
│ 3. 内核空间 (Kernel Space - Power Supply Class)                       │
│    - Linux 内核电源供应子系统 (power_supply 核心库)                      │
│    - /sys/class/power_supply/battery/ (包含温度、电压、电量、状态等)      │
└───────────────────────────────────┬────────────────────────────────────┘
                                    │ I2C / SPI / ADC
┌───────────────────────────────────▼────────────────────────────────────┐
│ 4. 物理硬件 (Hardware Layer)                                           │
│    - PMIC (电源管理芯片) & 电量计芯片 (Fuel Gauge)                        │
│    - 锂电池 & 充电芯片 (Charger IC)                                     │
└────────────────────────────────────────────────────────────────────────┘
```

## 二、 内核底层接口：`power_supply` 子系统

Linux 内核中，所有关于供电、电池、充电器的驱动，都注册在内核的 **`power_supply`** 子系统中。在用户空间，它们表现为 `/sys/class/power_supply/` 目录下的虚拟节点。

在典型 Android 设备上，通常包含以下三个关键供电目录：
* `/sys/class/power_supply/ac/`（交流电充电器）
* `/sys/class/power_supply/usb/`（USB 接口供电）
* `/sys/class/power_supply/battery/`（电池本身）

而在 `/sys/class/power_supply/battery/` 下，有以下高频读取的只读/读写节点：
* **`capacity`**：当前剩余电量百分比（0 ~ 100）。
* **`temp`**：电池实时温度（单位通常是 0.1 摄氏度，如 352 代表 35.2℃）。
* **`voltage_now`**：电池当前电压（单位是微伏 $\mu V$）。
* **`current_now`**：电池当前的充/放电物理电流（单位是微安 $\mu A$）。**正值通常代表充电，负值代表放电。**
* **`status`**：充电状态（Charging / Discharging / Full / Not Charging）。

## 三、 硬件抽象层：Health HAL (AOSP 14)

在早期版本中，Android 依靠 C++ 编写的 `healthd` 守护进程解析 Sysfs。在 Android 14 中，Health HAL 已经完全实现 Binder 化，采用 AIDL 接口 **`android.hardware.health.IHealth`**。

### 1. 核心结构体 `HealthInfo`
底层的 Health HAL 定期读取 `/sys/class/power_supply/` 下的所有节点，并将其打包封装进 `HealthInfo` 结构体中，通过 Binder 回传给 Java 层：

```cpp
// 对应 HAL AIDL 定义
struct HealthInfo {
    BatteryStatus batteryStatus;        // 充电状态 (Charging/Discharging)
    BatteryHealth batteryHealth;        // 电池健康度 (Good/Overheat/Dead)
    boolean batteryPresent;            // 电池是否在位
    int32_t batteryLevel;              // 电量百分比
    int32_t batteryVoltageMillivolts;  // 电池电压 (mV)
    int32_t batteryTemperatureTenthsCelsius; // 电池温度 (0.1℃)
    String batteryTechnology;          // 电池技术 (如 Li-poly)
    ...
}
```

## 四、 核心 Java 服务：`BatteryService`

`BatteryService` 运行在 `system_server` 中，它的初始化和工作原理非常清晰：

### 1. 监听 Health HAL 状态回调
在 `BatteryService` 启动时，它会向 Health HAL 注册一个回调监听器：

```java
// base/services/core/java/com/android/server/BatteryService.java

public void onStart() {
    ...
    // 绑定 Health HAL，并注册回调
    mHealthServiceWrapper.registerCallback(new HealthInfoCallbackImpl());
    ...
}
```

一旦电量计芯片检测到电压、电流或电量发生变化，Health HAL 就会收到内核的 `Netlink` 广播，解析后调用 `HealthInfoCallbackImpl` 的回调方法，更新 `BatteryService` 内部的变量。

### 2. 发布粘性广播 `ACTION_BATTERY_CHANGED`
当电池属性（电量、温度、电压等）改变时，`BatteryService` 会调用 `sendIntentLocked()`。
这是一个 **粘性广播（Sticky Broadcast）**：`Intent.ACTION_BATTERY_CHANGED`。

```java
// 发送电池变更广播
Intent intent = new Intent(Intent.ACTION_BATTERY_CHANGED);
intent.addFlags(Intent.FLAG_RECEIVER_REGISTERED_ONLY | Intent.FLAG_RECEIVER_REPLACE_PENDING);
intent.putExtra(BatteryManager.EXTRA_LEVEL, mHealthInfo.batteryLevel);
intent.putExtra(BatteryManager.EXTRA_TEMPERATURE, mHealthInfo.batteryTemperatureTenthsCelsius);
intent.putExtra(BatteryManager.EXTRA_VOLTAGE, mHealthInfo.batteryVoltageMillivolts);
...
mContext.sendStickyBroadcastAsUser(intent, UserHandle.ALL);
```

由于该广播带有 `FLAG_RECEIVER_REGISTERED_ONLY`，因此只有动态注册的 Receiver 才能收到，防止静态注册导致大量后台进程在电量波动时被瞬间拉起（造成“广播风暴”）。

## 五、 智能充电保护与安全策略

为了防止电池老化、胀气以及热失控，Android 14 进一步增强了充电安全机制：

1. **温控充电熔断（Thermal Charging Limits）**：
   当电池温度过高（如达到 45℃），`BatteryService` 会触发警告，并联动 `ThermalManager`。底层充电驱动会直接调低充电电流；若温度达到危险阈值（如 50℃），系统会彻底关断充电回路，进入 `Not Charging` 状态。
2. **80% 充电上限保护（Battery Health Protection）**：
   许多现代手机在插着充电器过夜时，系统会自动将充电阈值限制在 80%。当电量到达 80% 后停止充电，只由外接电源供电，在用户起床前一小时再充满至 100%。这极大地降低了锂电池处于高电压满格状态下的物理损耗。
